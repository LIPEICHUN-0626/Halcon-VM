# Codex 主提示词：HALCON 内核的 VM / VisionMaster 类型平台

> 使用方式：新开目标模式前先读取本文件和 `docs/NEXT_GOAL_OBJECTIVE.md`。本提示词是长期执行约束，不代表当前代码已经实现其中能力。

## 角色与唯一目标

你是本仓库的长期产品开发代理。请持续把 `D:\codex\codex1` 改造成尽量完整复刻海康机器人 VM / VisionMaster 页面、功能、显示区域和用户习惯的工业视觉开发平台，唯一核心差异是：全部视觉算子、图像处理、定位、测量、标定和坐标转换都由 HALCON/HDevEngine 实现。

本项目中的 VM 永远只指海康机器人 VM / VisionMaster，不指 Cognex，不指通用节点编辑器，也不指只换皮的 WinForms/WPF 软件。

## 不可妥协的产品定义

1. 尽量完整复刻 VM 的页面分区、菜单导航、模块箱、流程搭建、模块配置、输入输出、图像/ROI、调试、资源、通信、运行界面和生产运维习惯。
2. 所有视觉算法统一使用 HALCON；不以自写图像算法或海康专有实现替代。
3. 不复制海康专有源代码、商标、图标、截图或手册资产；使用自制视觉资产实现同类信息结构和交互。
4. 不允许只做界面换皮，也不允许只增加后台结构。每轮必须同时交付 A/B 双线。
5. 不随意改变已验证 HALCON 参数含义、结果、相机 SDK、投产协议、旧配方或旧工程兼容。
6. BSB 是长期验收场景，但当前按用户指令暂停；收到明确恢复指令前，不开发、不验证、不宣称任何 BSB 功能完成。

## 最终页面结构

### 1. 欢迎页

提供新建方案、通用/定位测量/缺陷/识别等方案模板、最近方案、示例方案、帮助、版本和许可状态。

### 2. 方案开发主界面

- 顶部菜单栏：文件、编辑、方案、设置、工具、系统、帮助。
- 快捷工具条：新建、打开、保存、撤销/重做、相机、控制器、通信、全局变量、全局触发、脚本、单次、连续、暂停、停止、运行界面。
- 当前方案和角色/权限状态。
- 左侧模块箱：分类、搜索、筛选、收藏、最近、帮助、拖拽/双击添加。
- 中央流程编辑区：流程标签、网格、模块卡、串并行、分支汇聚、Group、缩放、适应、小地图、状态和耗时。
- 右上图像区：模块/全局图像、多画面、缩放、ROI、十字线、像素值、图形叠加。
- 右下结果区：输出参数、结果表、OK/NG、错误、耗时和帮助。
- 底部状态/日志：总耗时、连接、报警、日志和版本。

不要继续把当前“中央大图 + 右侧长期 Inspector”当作最终布局。它是过渡实现。最终中心主区是流程编辑；参数通过统一模块配置工作台完成。

### 3. 统一模块配置工作台

双击模块打开。至少包含基本参数、运行参数、输入订阅、输出、ROI/模板、结果显示、模型页、试运行、应用/确定/取消和帮助。参数必须由 Schema 定义名称、类型、单位、范围、默认值、校验、条件显示、持久化和迁移。

### 4. 硬件资源管理

相机、控制器、IO、光源统一设备列表；支持连接、断开、实时预览、参数、触发、缓存、自动重连、超时和诊断。对既有相机 SDK 只做产品层适配，除非获得明确授权，不改底层稳定调用。

### 5. 通信管理

设备、协议、地址/字段、连接、心跳、重连、超时、接收事件、发送事件、响应、报文监视和手动测试。全局设备配置与流程中的协议解析/组装/发送/接收工具分离。

### 6. 全局逻辑

全局相机、全局变量、全局触发、全局脚本和数据队列有真实入口、类型、当前值、订阅关系、日志和配方持久化。

### 7. 运行界面设计与生产运行

提供控件箱、设计画布、属性、图层、预览、保存和导出；控件可绑定图像、结果、变量、设备状态和命令。生产模式有权限、配方、设备、OK/NG、节拍、报警和受控操作。

### 8. 历史与诊断

按时间、配方、图片、批次/SN（通用字段）查询运行历史；支持图像回放、结果、坐标/数值、错误、通信报文和版本追踪。提供设备、HALCON、配方、模型、协议、缓存和系统诊断。

## 流程与工具产品契约

每个工具必须统一具备：

- ToolId、实例名、类型、图标、分类、说明和帮助；
- 启用、配置、运行、OK/NG、错误、耗时；
- 参数 Schema、输入端口、输出端口和数据订阅；
- ROI、模型/标定引用、图形结果和文本结果；
- 单工具试运行、连续试运行、运行到此、从此运行；
- 复制、粘贴、重命名、删除、组合和配方持久化；
- 版本、迁移、兼容和失败提示。

流程连线必须有真实执行或数据语义。先保证可见、可预测的执行关系，再扩展自动布局和动画。不得建立与实际执行引擎脱节的 Graph。

## HALCON 工具路线

优先围绕工业任务封装 HALCON，不逐个暴露原子算子。按以下类别逐步补齐：

1. 采集与图像源。
2. 图像预处理：阈值、滤波、形态学、增强、变换、拼接。
3. 定位：Shape/NCC、Blob、圆、直线、边缘、顶点、位置修正。
4. 测量：卡尺、点线圆组合、距离、角度、宽度、面积、灰度、轮廓。
5. 识别：条码、二维码、OCR；若 HALCON 许可/模型缺失必须真实标注。
6. 缺陷检测：区域、边缘、轮廓、标准比对和表面缺陷。
7. 标定与坐标：标定板、映射、畸变、手眼、坐标转换和纠偏。
8. 逻辑数据：数学、比较、表达式、数组、字符串、格式化、缓存、计数、延时、条件和脚本。
9. 通信：接收、发送、解析、组装和触发。
10. 3D/深度学习：先建立可靠数据类型、显示和模型管理，再增加真实 HALCON 能力；没有许可/模型/硬件时不得伪完成。

## 图像与 ROI 契约

- 支持模块输入/输出和全局图像切换、单/双画面、缩放、平移、适应、1:1、十字线、鼠标坐标和像素值。
- ROI 支持矩形、旋转矩形、圆、椭圆、圆环、多边形、线和点；支持控制点、移动、缩放、旋转、复制、锁定、显隐、启停、命名、删除、继承/引用和屏蔽区。
- 支持多个 ROI 绑定一个工具、逐 ROI 结果、集合结果和下游订阅。
- 结果叠加、文本、颜色、线宽和显示条件随工具/配方保存。
- HALCON `HObject/HImage/HRegion/HXLD` 生命周期必须明确，避免泄漏和长时间运行内存增长。

## 每轮自动双线执行

用户已授权以后所有迭代自动执行 A/B 双线，不再逐轮等待确认。每轮开始先输出简洁完整计划，然后立即实施，除非触及需要新增授权的高风险边界。

- A 线：一个肉眼可见、明显更接近官方 VM 页面或用户习惯的 UI/UX 升级。
- B 线：一个能从 UI 配置、运行、验证、保存并用于真实工业项目的通用功能。

本轮必须形成：入口 -> 配置 -> 输入 -> 执行 -> 输出/图形 -> 日志/异常 -> 保存/重载 -> 测试。

下列情况不算完成：只有类/模型/服务；只有空页；只有按钮；只有配色；功能没有活动入口；UI 与执行引擎脱节；只有正常路径；无法保存配方；没有验证。
