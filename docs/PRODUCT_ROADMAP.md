## 状态定义

- **已验证实现**：活动产品入口可用，代码存在且有运行/测试证据。
- **部分实现**：基础能力可用，但缺产品闭环或工业特性。
- **开发中**：已确认并正在本轮实施。
- **提议**：已进入路线图，尚未开始。
- **未实现**：无活动产品能力或只有不可达/内部结构。

## 官方 VM 对标后的路线纠正

官方手册界面和 VM5.0 发布资料确认：开发主界面应以中央流程编辑为主，右侧分为图像和结果；流程长期需要网格布局、分支汇聚、自动串/并连排列和 Group 折叠；模块配置应形成统一的基本参数、运行参数、输入订阅、ROI/模板和结果显示工作台。

S04 已把活动主分区纠正为 `左模块箱 + 中央流程 + 右上图像 + 右下结果/配置`；S05 已把 Blob、灰度、边缘和数值判定迁入统一模块配置工作台首版并建立实例参数持久化。下一阶段深化图像/ROI 的统一配置与逐区域结果；中央流程在真实调度模型完成前仍只宣称确定性线性执行，不把视觉连线冒充并行、汇聚或 Group。

## P0：S03 VM 图像工具栏 + 工具级多 ROI（已验证实现）

- 验证：两块白色区域合并 Blob 面积 `500`，停用一个图层后为 `100`；完整 `Blob → NumericJudge` 分别得到 OK/NG；JSON 与界面保存回读保持 2 图层/2 绑定；旧 `SearchRoi` 迁移为 1 图层/1 绑定；未绑定必需 ROI 被拒绝。
- 回归与视觉：Blob/灰度/边缘继续为 `3850 / 51.133 / 246.109`；79 个 XAML 事件入口无缺失；1920×1200 下两 ROI 同时叠加、中央工具栏和两处图层管理入口完整显示；Debug/Release 0 错误 0 警告，Release 启动通过。
