## 状态定义

- **已验证实现**：活动产品入口可用，代码存在且有运行/测试证据。
- **部分实现**：基础能力可用，但缺产品闭环或工业特性。
- **开发中**：已确认并正在本轮实施。
- **提议**：已进入路线图，尚未开始。
- **未实现**：无活动产品能力或只有不可达/内部结构。

## P0：当前 Sprint（开发中）

主题：**VM 顺序流程工作区 + BSB 二维点位补偿**。

- A 线：增加左侧工具箱、清晰的顺序流程实例区、选中工具 Inspector；使工具可添加、选择、启停、重命名、排序和独立运行。
- B 线：增加 BSB 二维点位补偿工具，支持手工/模板匹配输入、旋转中心、平移、旋转、缩放、点数与越界检查、输出表和配方保存。
- 稳定边界：现有 HALCON Shape/Blob/Gray/Edge 算法实现和参数语义不变；旧配方通过缺省映射继续读取。

## P1：VM 项目闭环基础

1. 统一工具参数 Schema 和输入输出订阅。
2. 相机设备管理页与活动 WPF 采集入口。
3. ROI 控制点编辑、旋转矩形、多 ROI 和工具级 ROI。
4. 配方版本、迁移、另存为、自动保存和崩溃恢复。
5. 运行历史持久化、SN/载具/批次维度和图片回放。
6. TCP 工业化：心跳、重连、超时、协议帧、字段映射、触发和回复。

## P2：BSB 核心能力

1. 双 Mark 管理与焊房复拍补偿。
2. 极柱点集、点位排序、数量和越界防呆。
3. 多模组缓存与 SN/载具号绑定。
4. 焊前到焊房、相机到机械轴、压脚、振镜和焊接坐标链。
5. PLC 握手、OK/NG 复位、地址配置和坐标批量发送。
6. 首件、补焊、焊接顺序与项目诊断。
7. CSV/MES 追溯。

## P3：平台化与运维

- Modbus TCP/RTU、Siemens S7、Omron CIP/EtherNet-IP、UDP、串口和自定义协议。
- 全局变量、表达式、数组、字符串、条件、缓存、计数、延时和脚本工具。
- 用户权限、生产模式、多语言、License、插件管理、数据库、MES 和系统诊断。
- 多相机、深度图和点云产品入口。

## 路线图维护规则

每轮结束按真实代码、编译、启动和功能验证结果更新；未从 UI 使用、不可独立测试或只有内部类的能力不得进入“已验证实现”。
