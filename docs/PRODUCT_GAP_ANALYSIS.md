# 产品差距分析

更新日期：2026-07-19
目标：海康 VM / VisionMaster 类型工业视觉平台体验 + HALCON 算法内核

代码核验基线：S15 已验证源码（2026-07-19 本地验证，S06-S15 批次 10/10 批量交付，新批次 0/10）
官方参照：`docs/VM_OFFICIAL_REFERENCE_SPEC.md`

## 0. 官方资料复核后的总判断

当前软件已经完成官方 VM 开发主分区、首批右侧常驻模块配置、真实 ROI/逐区结果、连续 Image/Region 跨工具订阅、图像上下文和单方案可靠生命周期，但**仍不接近完整海康 VM**。S15 已让中央卡片显示来自真实元数据/绑定/运行快照的 IN/OUT 端口、来源、值、消费者与状态，并交付首个双 Region 并/交/差/对称差汇聚；这比只有内部 `InputBindings` 更有用户价值。当前仍是纵向列表与确定性线性调度，没有 VM5 类型的二维连接路径、自动串/并连分层、真实分支调度、Group 或多流程；Shape/HDevelop、完整 ROI、数组、资源/权限/运行界面、完整工具目录和生产运维也未完成，因此只能称为 VM 化 HALCON 工作台的早期产品闭环。

本结论来自海康[VM 产品页](https://www.hikrobotics.com/cn/machinevision/visionmaster/)、[VM5.0 发布资料](https://www.hikrobotics.com/cn/publish/)和[官方 VM3D 用户手册](https://pinfo.hikrobotics.com/hkws/unzip/20240919112534_17943_doc/index.html)。VM3D 的通用框架用于确认 VM 家族页面结构，不用来冒充本项目已有 3D 能力。

## 1. 当前活动架构

- `App.xaml` 启动 `MainWindow.xaml`，因此 WPF 主窗口是唯一活动产品入口。
- `MainWindow` 直接持有图像、ROI、模板、通讯、配方、结果和运行状态，并以事件处理器驱动。
- `RecipeService` 已独立承担版本化 JSON、原子替换、单份备份、布局/最近记录和恢复副本；`MainWindow` 仍负责脏状态与用户提示，尚未形成完整工程域服务。
- `MainForm` 和 `TemplateCreateForm` 仍参与编译，但没有活动启动引用；其中相机、Mask、显示框编辑和结果筛选等能力对 WPF 用户不可达。
- 已有统一工具实例、类型化 IN/OUT 端口、Number/Image/Region 订阅、ROI 图层、连续图像/区域工具链和首个双 Region 集合运算；输出使用托管 HImage/HRegion 并在重跑/删除/关闭时释放。仍没有数组、XLD/几何/姿态数据和完整图像处理目录。
- 活动流程以纵向节点卡显示确定性执行顺序；S15 卡内端口和直接上下游高亮来自真实绑定，双输入汇聚语义真实，但尚无二维连接路径、依赖分层自动布局、真实多流程/并行/Group 或断点模型。
- 右侧常驻模块配置工作台覆盖本地图像源、图像通道、图像滤波、阈值分割、Region 形态学、Region 特征筛选、Region 集合运算、Blob、Gray、Edge、NumericJudge 的基本/运行/输入/ROI/结果/帮助、草稿、应用/撤销和试运行；Shape/HDevelop 和模型训练仍为兼容弹窗。

## 3. 只有内部结构或不可达的实现

1. `HalconImageService` 的相机采集方法只由不可达 `MainForm` 使用，当前 WPF 用户无法配置和采集。
2. `MainForm` 内较丰富的模板 Mask、显示框控制点、相机和结果过滤没有迁移到活动 WPF。
3. `RoiService` 没有活动引用。
4. `InspectionResultStore.Query` 只在不可达旧窗口中使用；WPF 结果区不能查询筛选。
5. WPF 保存 `LastImageDirectory` 和 `TemplateOptions`，但加载路径没有完整消费这些信息。
6. WPF 原图副本和首条模块 Image 输入/输出已进入上下文切换，但多数视觉工具尚无独立处理中间图/结果图 Image 输出。
7. `RoiService`、旧 WinForms 的 Mask/显示框控制点和部分相机入口没有迁移到活动 WPF，不能计入当前产品能力。
8. 工具流、实例参数、端口和 ROI 已有活动 UI，但只有十三类工具以及首个双 Region 汇聚；数组、XLD/几何数据、真实连接画布和完整视觉工具目录仍缺，距离 VM 的完整模块契约很远。

## 4. 用户重点问题逐项回答

### UI 是否真的接近海康 VM

主分区和首批调试路径已经明显接近，S15 又让端口、来源、消费者和双输入汇聚在流程卡可见。仍不能称为完整复刻：流程仍是纵向列表和线性调度，没有 VM5 类型的二维连接路径、自动分层、分支/Group/多流程；Shape/HDevelop、完整 ROI、方案树/模板、资源、权限、运行界面和大量工具目录也尚未实现。

### 主要参数是否仍依赖弹窗

**首批主要视觉参数已不再依赖弹窗，但迁移尚未完成。** S10-S15 已把本地图像源、图像通道、图像滤波、阈值分割、Region 形态学、Region 特征筛选、Region 集合运算、Blob、灰度、边缘和数值判定迁入右侧常驻配置。HDevelop、Shape 模板训练和模型管理仍使用兼容弹窗，通用 Schema 与连续试运行也未完成。

### 流程工具区是否符合 VM 习惯

S04/S12 已补齐线性调试、区段运行和拖拽插入，S15 增加真实端口、消费者、依赖高亮和双输入汇聚。流程数据语义明显改善，但仍缺二维连接路径、依赖分层自动排列、复制粘贴、真实分支调度、Group 和多流程。

### 图像区和 ROI 是否完善

S03/S06/S07 的 ROI 图层、逐区结果和 Blob Region，S09-S12 的图像上下文、连续 Image 链与阈值 Region，S13 的图像放大/叠加和形态学 Region，以及 S14 的旋转矩形/多边形直接编辑与特征筛选都是真实闭环，但仍不完善。当前只有单图像视口和单行结果叠加摘要；还缺圆环/椭圆/线/点、屏蔽区、位置修正、十字线、多画面、完整叠加层清单、完整图像处理目录和多数工具的中间图/结果图输出。

### 通信、计算、坐标和 BSB 是否有真实入口

- 通信：有真实 TCP Client/Server 页、手动收发和自动结果入口，但缺工业通信管理。
- 计算：有真实数值判定工具和 Number 订阅，但没有通用数学、逻辑、数组、字符串、表达式和脚本工具集。
- 坐标/标定：没有活动流程工具或配置入口。
- BSB：没有活动工具，且当前按用户指令暂停。

### 是否有大量只有内部结构、没有用户价值的实现

存在，最明显是相机采集服务、旧 `MainForm` 的相机/Mask/结果筛选、`RoiService` 和 `InspectionResultStore.Query`。它们对活动 WPF 用户不可达，因此只能作为潜在迁移资产，不能计入已交付产品能力。

## 5. 已识别的具体风险

- 无真实 Shape Model 模板样本，模板训练/保存/加载的本轮回归证据不足。
- 训练后未另存 `.shm` 的模板没有持久路径，保存配方时无法完整恢复。
- 配方目前只有单份 `.bak` 和一个活动恢复副本，没有版本历史、签名、权限、自动换型事务或跨机资源打包；`.jw/.handle` 仍无证据。
- TCP 以每次 `Read` 直接解码消息，没有工业协议分帧；没有连接超时、心跳和自动重连。
- 连续运行仍由 UI `DispatcherTimer` 触发，HALCON 工具在 UI 线程同步执行；已支持步骤间暂停/停止/超时和防重入，但不能中断单个长耗时算子，尚未形成生产线程、硬取消和完整节拍治理模型。
- 结果图像快照全部驻留内存，长时间运行存在内存增长风险。
- 当前没有真实 `.jw/.handle`、PLC 地址表、BSB 标定和协议样本，相关兼容性仍缺权威验证材料。

## 6. 当前消差策略

S04-S15 已完成主分区、常驻配置、ROI/Region、可靠配方、连续 Image 数据链、分类模块箱、图像调试区、真实端口可视化和首个双 Region 汇聚。下一步执行 S16：把真实绑定映射为二维连接路径和依赖分层自动布局首版，并新增可连续输出托管 Image 的 HALCON 图像增强工具。资源管理、通信、全局逻辑、运行界面和历史诊断随后推进；相机 WPF 入口涉及既有相机 SDK 边界，实施前按约束单独确认。BSB 保留但暂停。

每轮以代码入口、用户路径、独立测试、编译和启动为完成证据；BSB 恢复开发后必须增加对应场景验证。没有证据的条目保持“未实现”或“部分实现”。
