# 产品差距分析

更新日期：2026-07-13  
目标：海康 VM / VisionMaster 类型工业视觉平台体验 + HALCON 算法内核

代码核验基线：S04 已验证源码（基于 `ebf1c0b`）
官方参照：`docs/VM_OFFICIAL_REFERENCE_SPEC.md`

## 0. 官方资料复核后的总判断

当前软件已经完成官方 VM 开发主分区首版，但**仍不接近完整海康 VM**。活动 WPF 现在以中央网格顺序流程为核心，右侧上/下分别为 HALCON 图像/ROI 和模块结果/配置，顶部菜单/快捷工具条及底部状态/历史可见；这修复了 S01-S03 的主信息架构反向问题。仍缺统一模块配置、真实分支/汇聚/Group、多流程、资源与运行界面、完整工具目录和生产运维，因此只能称为 VM 化 HALCON 工作台的早期产品闭环。

本结论来自海康[VM 产品页](https://www.hikrobotics.com/cn/machinevision/visionmaster/)、[VM5.0 发布资料](https://www.hikrobotics.com/cn/publish/)和[官方 VM3D 用户手册](https://pinfo.hikrobotics.com/hkws/unzip/20240919112534_17943_doc/index.html)。VM3D 的通用框架用于确认 VM 家族页面结构，不用来冒充本项目已有 3D 能力。

## 1. 当前活动架构

- `App.xaml` 启动 `MainWindow.xaml`，因此 WPF 主窗口是唯一活动产品入口。
- `MainWindow` 直接持有图像、ROI、模板、通讯、配方、结果和运行状态，并以事件处理器驱动。
- `MainForm` 和 `TemplateCreateForm` 仍参与编译，但没有活动启动引用；其中相机、Mask、显示框编辑和结果筛选等能力对 WPF 用户不可达。
- 已有首版统一工具实例、中央网格顺序流程、类型化 IN/OUT 端口、数值订阅、ROI 图层和工具级多 ROI 绑定；仍没有统一参数 Schema、跨工具图像/区域订阅、多输入绑定或 BSB 模型。
- 活动流程以节点卡和真实前后序连线显示确定性线性执行，支持运行当前、运行到此、从此运行、连续、暂停/停止、间隔、超时和遇 NG 停止；尚无真实多流程、并行、分支、汇聚、Group 或断点模型。
- 参数区由 Shape/Blob/Gray/Edge/HDevelop/NumericJudge 六组手写 Panel 组成；不存在统一模块配置 Schema 或 VM 类型模块配置工作台。

## 3. 只有内部结构或不可达的实现

1. `HalconImageService` 的相机采集方法只由不可达 `MainForm` 使用，当前 WPF 用户无法配置和采集。
2. `MainForm` 内较丰富的模板 Mask、显示框控制点、相机和结果过滤没有迁移到活动 WPF。
3. `RoiService` 没有活动引用。
4. `InspectionResultStore.Query` 只在不可达旧窗口中使用；WPF 结果区不能查询筛选。
5. WPF 保存 `LastImageDirectory` 和 `TemplateOptions`，但加载路径没有完整消费这些信息。
6. WPF 保存原图副本，但没有原图/处理中间图/结果图切换入口。
7. `RoiService`、旧 WinForms 的 Mask/显示框控制点和部分相机入口没有迁移到活动 WPF，不能计入当前产品能力。
8. 工具流、端口和 ROI 已有内部模型，但只有六类工具、一个 Number 输入和线性执行；距离 VM 的通用模块契约仍很远。

## 4. 用户重点问题逐项回答

### UI 是否真的接近海康 VM

主分区已经明显接近：中心是网格流程，右侧上方是图像/ROI、下方是模块结果/配置，顶部和底部运行状态完整。仍不能称为完整复刻，因为流程只有线性语义，模块配置仍是手写 Inspector，方案树、资源管理、运行界面和大量工具目录尚未实现。

### 主要参数是否仍依赖弹窗

Blob、灰度、边缘、HDevelop、数值判定和部分 Shape 参数已可在活动 Inspector 配置，这是用户价值；但模板训练仍依赖 `TemplateCreateForm`，且当前 Inspector 是每类工具手写控件，不具备 VM 类型统一基本/运行/结果/ROI/模型配置体系。问题不是“有无弹窗”，而是缺统一模块配置工作台。

### 流程工具区是否符合 VM 习惯

S04 已补齐网格背景、流程标签、真实顺序连线、缩放/适应、运行当前/到此/从此、连续间隔、暂停/停止、流程超时和遇 NG 停止，线性调试习惯已形成闭环。仍缺拖拽自由布局、复制粘贴、真实分支/汇聚、自动排列、Group 和多流程调度。

### 图像区和 ROI 是否完善

S03 的多 ROI 图层、工具绑定和 HALCON Region 并集是真实闭环，但仍不完善。缺旋转矩形、控制点、复制/锁定、圆环/椭圆/线/点、屏蔽区、位置修正、逐 ROI 输出、十字线/像素值、模块/全局图像和原图/中间图/结果图切换。

### 通信、计算、坐标和 BSB 是否有真实入口

- 通信：有真实 TCP Client/Server 页、手动收发和自动结果入口，但缺工业通信管理。
- 计算：有真实数值判定工具和 Number 订阅，但没有通用数学、逻辑、数组、字符串、表达式和脚本工具集。
- 坐标/标定：没有活动流程工具或配置入口。
- BSB：没有活动工具，且当前按用户指令暂停。

### 是否有大量只有内部结构、没有用户价值的实现

存在，最明显是相机采集服务、旧 `MainForm` 的相机/Mask/结果筛选、`RoiService` 和 `InspectionResultStore.Query`。它们对活动 WPF 用户不可达，因此只能作为潜在迁移资产，不能计入已交付产品能力。

## 5. 已识别的具体风险

- 无真实 Shape Model 模板样本，模板训练/保存/加载的本轮回归证据不足。
- 训练后未另存 `.shm` 的模板没有持久路径，保存配方时无法完整恢复。
- TCP 以每次 `Read` 直接解码消息，没有工业协议分帧；没有连接超时、心跳和自动重连。
- 连续运行仍由 UI `DispatcherTimer` 触发，HALCON 工具在 UI 线程同步执行；已支持步骤间暂停/停止/超时和防重入，但不能中断单个长耗时算子，尚未形成生产线程、硬取消和完整节拍治理模型。
- 结果图像快照全部驻留内存，长时间运行存在内存增长风险。
- 当前没有真实 `.jw/.handle`、PLC 地址表、BSB 标定和协议样本，相关兼容性仍缺权威验证材料。

## 6. 当前消差策略

S04 已完成官方 VM 主分区首版和真实线性运行策略。下一步执行 S05：统一模块配置工作台、实例参数 Schema、同类型多实例与旧参数迁移；之后按图像/Region 订阅、ROI/结果、方案/配方、资源、通信、全局逻辑、运行界面和历史诊断推进。相机 WPF 入口涉及既有相机 SDK 边界，实施前按约束单独确认。BSB 保留但暂停。

每轮以代码入口、用户路径、独立测试、编译和启动为完成证据；BSB 恢复开发后必须增加对应场景验证。没有证据的条目保持“未实现”或“部分实现”。
