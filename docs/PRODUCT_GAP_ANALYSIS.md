# 产品差距分析

更新日期：2026-07-13  
目标：海康 VM / VisionMaster 类型工业视觉平台体验 + HALCON 算法内核

## 1. 当前活动架构

- `App.xaml` 启动 `MainWindow.xaml`，因此 WPF 主窗口是唯一活动产品入口。
- `MainWindow` 直接持有图像、ROI、模板、通讯、配方、结果和运行状态，并以事件处理器驱动。
- `MainForm` 和 `TemplateCreateForm` 仍参与编译，但没有活动启动引用；其中相机、Mask、显示框编辑和结果筛选等能力对 WPF 用户不可达。
- 已有首版统一工具实例、顺序流程、类型化 IN/OUT 端口和数值订阅；仍没有统一参数 Schema、图像/区域订阅、多输入绑定或 BSB 模型。

## 3. 只有内部结构或不可达的实现

1. `HalconImageService` 的相机采集方法只由不可达 `MainForm` 使用，当前 WPF 用户无法配置和采集。
2. `MainForm` 内较丰富的模板 Mask、显示框控制点、相机和结果过滤没有迁移到活动 WPF。
3. `RoiService` 没有活动引用。
4. `InspectionResultStore.Query` 只在不可达旧窗口中使用；WPF 结果区不能查询筛选。
5. WPF 保存 `LastImageDirectory` 和 `TemplateOptions`，但加载路径没有完整消费这些信息。
6. WPF 保存原图副本，但没有原图/处理中间图/结果图切换入口。

## 4. 已识别的具体风险

- 无真实 Shape Model 模板样本，模板训练/保存/加载的本轮回归证据不足。
- 训练后未另存 `.shm` 的模板没有持久路径，保存配方时无法完整恢复。
- TCP 以每次 `Read` 直接解码消息，没有工业协议分帧；没有连接超时、心跳和自动重连。
- 连续运行基于 UI DispatcherTimer，尚未形成生产线程、取消和节拍治理模型。
- 结果图像快照全部驻留内存，长时间运行存在内存增长风险。
- 当前没有真实 `.jw/.handle`、PLC 地址表、BSB 标定和协议样本，相关兼容性仍缺权威验证材料。

## 5. 当前消差策略

首版 VM 工具箱、顺序流程、Inspector、类型化端口和数值订阅已落地。下一阶段继续优先视觉软件 UI、操作习惯和通用功能：活动 WPF 相机入口、ROI 编辑、实例级参数 Schema、图像/区域订阅、配方版本和 TCP 工业化。BSB 需求保留但暂停，收到明确指令后再实施，避免提前堆叠不可用的专用结构。

每轮以代码入口、用户路径、独立测试、编译和启动为完成证据；BSB 恢复开发后必须增加对应场景验证。没有证据的条目保持“未实现”或“部分实现”。
