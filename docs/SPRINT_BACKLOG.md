## 当前 Sprint：S01 VM 顺序流程工作区 + BSB 二维点位补偿

状态：**开发中（计划已确认）**

### A 线：UI/UX 升级

当前问题：活动 WPF 主界面只有中央图像和右侧静态页签，五个检测工具由复选框固定启停，不符合 VM 的工具箱、顺序流程和 Inspector 工作方式。

计划交付：

1. 左侧增加分类工具箱，首批包含现有视觉工具与 `BSB 专用工具/BSB 点位补偿`。
2. 增加始终可见的顺序流程实例区，显示实例名、类型、启用、配置、结果、耗时和错误。
3. 支持从工具箱添加、删除、重命名、启停、上移、下移、运行当前和全流程运行。
4. 选中工具后右侧 Inspector 显示该实例的参数、输入输出和结果。
5. 保留中央 HALCON 图像区和底部日志/结果区，重新分配工作区比例。
6. 现有 Shape/Blob/Gray/Edge/HDevelop 执行方法通过适配接入，不改变算法和参数语义。

用户路径：工具箱双击或点击添加 -> 在流程区选择实例 -> 右侧配置 -> 运行当前/全流程 -> 图像与输出更新 -> 日志记录 -> 保存配方。

### B 线：实际工业功能

工具：`BSB 点位补偿`

参数：

- 输入来源：手工点集或 Shape Model 匹配结果；
- 旋转中心 X/Y；
- 平移 X/Y；
- 旋转角度（度）；
- 缩放比例；
- 期望点数；
- X/Y 允许范围；
- 输出小数位；
- 是否显示图形叠加。

输入：带 ID 的二维点集，产品坐标约定 `X=Column`、`Y=Row`。  
输出：原始点、补偿点、数量、OK/NG、错误信息和耗时。

运行逻辑：围绕指定中心缩放和旋转，再平移；拒绝无效数值、非正比例、点数不符和越界输出。结果进入流程摘要、右侧输出、日志和总结果 JSON。

独立验证：提供 BSB 示例点集和“运行当前”。已知用例 `(10,0)`、中心 `(0,0)`、比例 `2`、角度 `90°`、偏移 `(100,200)`，期望 `(100,220)`。

### BSB 价值

- 动作流：覆盖焊房 Mark 复拍后的点位补偿环节。
- 数据流：承接模板匹配中心或手工点集，并为后续 PLC/焊接输出提供标准点集。
- 坐标流：建立首个可验证的相机点到补偿点闭环，为标定矩阵和振镜转换奠定产品入口。

### 兼容、风险与回退

- 旧配方仍按现有布尔字段生成默认流程；新增字段缺失时使用安全默认值。
- 不修改 Shape Model、Blob、灰度、边缘和 HDevelop 算法实现。
- `MainForm`/`TemplateCreateForm` 暂时保留编译，不作为活动入口。
- 主要风险：WPF 布局变化、旧配方映射、HALCON Row/Column 与 X/Y 混淆、点位叠加坐标系误用。
- 回退点：Git 基线提交 `858924e` 和本 Sprint 文档基线提交。

### 验收标准

- UI 肉眼可见工具箱、顺序流程区和 Inspector。
- BSB 工具可从 UI 添加、配置、独立运行、保存和重新加载。
- 流程顺序真实影响执行顺序，运行状态、OK/NG、耗时和错误可见。
- 已知变换用例通过；点数和越界 NG 用例通过。
- Debug/Release x64 编译通过，程序启动通过。
- 现有图片、ROI、模板匹配和旧格式配方回读不回归。
- 产品文档按实际完成情况更新并提交 GitHub。
