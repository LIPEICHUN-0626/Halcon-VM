## 可直接使用的目标

进入长期产品目标模式，持续把 `D:\codex\codex1` 改造成尽量完整复刻海康机器人 VM / VisionMaster 页面、功能、显示区域和用户习惯的工业视觉开发平台，唯一核心差异是全部视觉算子、图像处理、定位、测量、标定和坐标转换均由 HALCON/HDevEngine 实现。

开始前完整阅读 `AGENTS.md`、`docs/VM_OFFICIAL_REFERENCE_SPEC.md`、`docs/CODEX_MASTER_PROMPT_VM_REPLICA.md`、`docs/PRODUCT_GOAL.md`、`docs/PRODUCT_ROADMAP.md`、`docs/SPRINT_BACKLOG.md`、`docs/PRODUCT_GAP_ANALYSIS.md` 和 `docs/BSB_REQUIREMENTS.md`，并以实际活动代码和可复现验证为准，不得只依赖历史总结。

永久执行规则：

1. VM 始终只指海康机器人 VM / VisionMaster；不做 Cognex、通用 Graph 或只换皮的软件。
2. 以海康官方产品页、VM5.0 发布资料和官方用户手册为产品参照；尽量完整复刻信息架构、主工作区、流程搭建、模块配置、图像/ROI、输入输出订阅、调试运行、资源管理、通信、运行界面、生产运维和工具目录。
3. 不复制海康专有代码、商标和美术资产；使用自制资产实现同类页面与交互。
4. 每轮自动执行 A/B 双线，不再等待逐轮确认：A 线必须有肉眼可见的 VM UI/UX 升级；B 线必须新增可从 UI 配置、运行、保存和验证的真实通用工业功能。
5. 每轮先输出完整计划，然后立即实施；只有恢复 BSB、改变 HALCON 算法含义、触碰相机 SDK、投产协议、旧工程格式或其他高风险边界时才暂停请求授权。
6. BSB 是长期验收场景，但当前按用户指令暂停；收到明确恢复指令前不开发、不验证、不宣称完成。
7. 不得用内部类、空页面、假按钮、Graph 动画、编译通过或模拟数据冒充产品完成。
8. 每轮完成入口、配置、输入、执行、输出/图形、日志/异常、配方保存/回读和测试闭环；执行 Debug/Release x64 编译、启动、HALCON 回归、旧配方兼容、异常路径和高 DPI UI 验证。
9. 每轮按真实代码更新产品目标、路线图、BSB 基线、Sprint、Gap 分析和必要的官方规范。工作区文件即时保存；每个可独立验证的小交付在 `docs/GIT_BATCH_LOG.md` 记 1 次，累计 10 次后统一提交并推送 GitHub `origin/main`，随后归零。除用户明确要求或高风险回退点外，不足 10 次不频繁上传。
10. 长期目标不会因单个 Sprint 完成而结束；只有完整 VM 产品验收矩阵全部满足后才可标记完成。
