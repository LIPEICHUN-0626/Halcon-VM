# BSB 电池产线需求基线

更新日期：2026-07-19
状态：长期核心验收场景；按用户指令暂缓实施

2026-07-19 S15 只更新通用 VM 流程端口/依赖可视化与 HALCON Region 集合运算，不恢复 BSB 排期。S15 代码仍没有可从活动 WPF 使用的 BSB 工具、现场坐标链、PLC 握手或追溯闭环，本轮按用户指令未实施/未验证 BSB；下述状态全部保持不变。只有用户明确发出恢复 BSB 的指令后，才从本基线选择 BSB A/B Sprint。

## 1. 场景定位

BSB 不是演示模板，而是检验本产品能否用于真实电池产线视觉项目的长期场景。产品必须同时覆盖动作流、数据流、坐标流、设备关系、通讯握手、防呆和追溯。

## 2. 动作流

上使能 -> 工站选择 -> 换型 -> 清空缓存 -> 载具到位 -> 读取载具号 -> 扫码绑定 SN -> 焊前 Mark 拍照 -> 极柱寻址 -> 数据缓存 -> 产品进入焊房 -> 焊房调用数据 -> 焊房 Mark 复拍 -> 压脚确认 -> 坐标计算 -> 振镜偏移调用 -> 焊接坐标输出 -> 焊接完成反馈 -> OK/NG -> CSV/MES。

## 3. 数据流

PLC -> 相机/视觉 -> HALCON 工具流程 -> 数据缓存 -> 坐标转换 -> 防呆 -> PLC/机器人/振镜/焊接软件 -> 日志和追溯。

每个关键数据包至少携带：工站、配方、载具号、SN、模组号、拍照序号、工具版本、时间、输入点、输出点、判定、错误码和关联图片。

## 4. 坐标流

焊前相机坐标 -> 焊前实际坐标 -> 焊房相机坐标 -> 机械轴坐标 -> 压脚坐标 -> 振镜坐标 -> 焊接坐标。

永久约定：HALCON 图像坐标 `Column` 对应产品层 `X`，`Row` 对应产品层 `Y`；涉及物理坐标时必须显式显示坐标系名称、单位、标定版本和转换方向。

## 5. 设备关系

| 设备/系统 | 主要职责 | 产品必须提供 |
| --- | --- | --- |
| PLC | 节拍、触发、工站、换型、握手、结果 | 设备状态、地址/字段配置、手动测试、日志、超时、复位 |
| 相机 | 焊前/焊房 Mark、极柱和复拍图像 | 设备管理、采集状态、单帧/连续、参数与图像关联 |
| HALCON 流程 | 定位、测量、点集和图形结果 | VM 工具实例、参数、输入输出、ROI、独立运行 |
| 机器人/机械轴 | 机械坐标和运动 | 坐标系、范围、防呆和发送确认 |
| 振镜/焊接软件 | 偏移、焊点、顺序和焊接反馈 | 标定版本、点集批发、应答、重试和追溯 |
| 扫码/MES | SN、载具、工单和追溯 | 绑定、校验、缓存、补传和查询 |

## 7. 当前现场痛点假设

这些条目需要在获得真实 PLC 地址表、协议和样本数据后进一步确认：

- 坐标系和单位在多个软件间容易混淆，转换方向难追踪。
- 点位数量、顺序或范围异常若发现太晚，会进入焊接环节。
- 焊前与焊房数据依赖载具/SN 对应，缓存错配风险高。
- 设备握手和复位失败时，现场缺少统一诊断视图。
- 换型后配方、标定、PLC 字段和点集版本容易不一致。
- 日志若不能按 SN/载具关联，问题难以复盘。

## 8. BSB 功能统一交付要求

每个 BSB 功能必须同时交付 VM 工具节点、右侧参数、输入输出、图形/点位显示、日志、异常、独立测试、示例流程和配方持久化。只有计算类或只有窗口不算完成。
